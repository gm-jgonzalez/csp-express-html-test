CSP Violation Test Cases

Selecciona uno de los siguientes escenarios para generar reportes CSP específicos:

  • 1. XSS (Script externo)

    Prueba la carga de scripts desde dominios no autorizados.

  • 2. XSS (Script inline)

    Prueba la ejecución de scripts inline sin nonce.

  • 3. Exfiltración de datos

    Prueba intentos de conexión a dominios externos no autorizados.

  • 4. Uso de eval()

    Prueba el uso de eval() cuando no está permitido.

  • 5. Inyección de estilo malicioso

    Prueba la carga de estilos desde dominios no autorizados.

  • 6. Iframe malicioso

    Prueba la carga de iframes desde dominios no autorizados.

  • 7. XSS basado en URLs data

    Prueba el uso de data: URLs para evitar restricciones.

  • 8. Worker malicioso

    Prueba la carga de web workers desde dominios no autorizados.

  • 9. Manejo incorrecto de MIME-type (Working on it)

    Prueba el uso de recursos con tipo MIME incorrecto.

  • 10. Clickjacking (Working on it)

    Prueba la carga de frame ancestors.
